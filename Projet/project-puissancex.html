Jeu du Puissance X

Jeu de puissance 4 réglable, on peut régler le nombre de lignes et de colonnes, le nombre de jetons pour gagner. Il est possible de jouer contre un bot ou contre un autre joueur.

Durée : 8 semaines Type : Projet universitaire Année : 2025

Présentation du projet

Puissance X est un jeu de société classique où deux joueurs s'affrontent pour aligner un nombre choisi de jetons (de 3 à 6) de leur couleur dans une grille. Le projet a été réalisé en groupe de cinq personnes dans le cadre d'un projet universitaire, et a été développé en Java avec l'IDE IntelliJ IDEA et partagé grâce à git. Le jeu permet de régler la taille de la grille (nombre de lignes et de colonnes) ainsi que le nombre de jetons à aligner pour gagner. Il est possible de jouer contre un bot ou contre un autre joueur.


Structuration et code

La figure 1 représente l'éditeur de code Visual Studio Code avec le fichier d'un bot ouvert, le conditionIA, au sein du projet PuissanceX.

À gauche, nous avons l'arborescence du projet, qui contient les fichiers nécessaires au fonctionnement de l'application. Pour trier les fichiers, j'ai choisi de les regrouper par dossier, en respectant le paradigme MVC. Le paradigme MVC (Modèle-Vue-Contrôleur) est une architecture logicielle qui sépare les préoccupations d'une application en trois composants principaux : le Modèle (qui gère les données et la logique métier), la Vue (qui gère l'affichage et l'interface utilisateur) et le Contrôleur (qui gère les interactions entre le Modèle et la Vue). Cela permet de mieux organiser le code et de le rendre plus lisible.

À droite, nous avons un extrait du code source de l'application, qui est écrit en Java. Il y a également une bibliothèque nommée Boardifier qui permet de manipuler la grille de jeu. Nous pouvons également voir les commentaires qui expliquent le fonctionnement de l'IA. Dans l’extrait de code affiché, on observe la classe ConditionIA, qui est une IA pour le jeu Puissance X. J'ai créé cette classe qui implémente une stratégie de jeu basée sur des conditions spécifiques pour déterminer le meilleur coup à jouer. Elle analyse la grille de jeu et évalue la meilleure position pour placer un jeton, en tenant compte des règles du jeu et des positions des jetons adverses. Cette approche permet à l'IA de prendre des décisions stratégiques et de jouer des coups efficaces pour gagner, tout en minimisant le risque de perdre.
Cependant cette IA n'est pas très intelligente, car elle regarde les coups possibles lors de ce tour, et du prochain tour. Elle ne regarde pas les coups possibles dans le futur, ce qui la rend vulnérable à des stratégies plus avancées.

Ce projet m’a permis de mieux comprendre la programmation orientée objet, la structuration de projets Java, et l’utilisation de bibliothèques externes. Il m’a également familiarisé avec la création d’interfaces utilisateur simples et la gestion des interactions entre les composants du jeu. Cependant, il reste encore des améliorations à apporter, notamment en ce qui concerne l'intelligence de l'IA. De plus j'ai pu avoir des problèmes sur le fait que l'IA ne place pas le jeton au bon endroit, ce qui a nécessité de revoir la logique de placement des jetons.

Configuration

La figure 2 représente ce qui est affiché sur la console lors de l'exécution de l'application PuissanceX.

En haut, nous avons les lignes de texte qui sont affichées pour informer l'utilisateur sur les paramètres du jeu. Et en bas, nous avons les options de configuration du jeu. Ces lignes de texte, générées par le programme Java, permettent de configurer le jeu avant de commencer à jouer.
Le choix 1 permet de lancer le jeu avec la configuration écrite au-dessus.
Le choix 2 permet de modifier le type de jeu (contre un bot ou contre un autre joueur ou encore faire un match entre deux bots).
Le choix 3 permet de modifier le nombre de lignes et de colonnes de la grille (de 3 à 6 ).
Le choix 4 permet de modifier le nombre de jetons à aligner pour gagner (de 3 à 6).
Le choix 5 permet de quitter le jeu.


Nous avons choisi de faire une interface en console pour simplifier le développement et nous concentrer sur la logique du jeu. Nous avons utilisé des boucles et des conditions pour gérer les choix de l'utilisateur et afficher les informations nécessaires.


Jeu

La figure 3 illustre le contenu affiché dans le terminal lors de l’exécution de l’application PuissanceX. Elle montre le jeu de plateau, avec les jetons des joueurs et le plateau.

Dans la figure 3, nous pouvons voir le jeu en cours, avec comme configuration :

  • Nombre de lignes : 5
  • Nombre de colonnes : 10
  • Jetons à aligner pour gagner : 6
  • Type de jeu : contre un bot
La partie est en cours, nous pouvons voir, en haut, le numéro des colonnes, ainsi que la personne qui doit jouer.
Au centre, nous avons le plateau de jeu.
En bas, nous avons un message qui indique à l'utilisateur de choisir une colonne pour placer son jeton. Et le bot joue automatiquement après le joueur.

Nous avons utilisé des boucles pour vérifier les entrées de l'utilisateur et gérer les tours des joueurs. Nous avons aussi utilisé des conditions pour gérer les choix de l'utilisateur et afficher les informations nécessaires. Cela a pu nous poser des problèmes, notamment contre les bots, qui affichent des jetons supplémentaires dans la grille, ce qui a nécessité de revoir le code des bots à plusieurs reprises.

Ce projet m’a permis de mieux comprendre la gestion des entrées/sorties en Java, la création d'interfaces utilisateur simples, et la gestion des interactions entre les joueurs. Il m’a également familiarisé avec la création de jeux en console et la gestion des règles du jeu.
